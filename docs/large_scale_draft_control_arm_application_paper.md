# 面向大规模可制造控制臂结构优化的神经网络多尺度有限元与拔模约束方法

> 应用型论文草稿。图、补充算例、最终数值表格、参考文献格式待后续补齐。

## 摘要

针对汽车控制臂等薄壁、孔系和局部安装区域复杂的承载构件，传统拓扑优化在工程应用中常面临三个矛盾：一是大规模三维有限元模型的求解代价高，二是优化结果难以直接满足拔模与连通制造约束，三是局部细观材料分布或多尺度形函数引入后会显著加剧线性系统病态性。本文围绕一个约一千万位移自由度、细密度单元数约 4.12 亿的控制臂算例，构建了一套面向超算环境的大规模可制造结构优化流程。该流程采用神经网络预测单元内部多尺度形函数，以 5 x 5 x 5 细胞密度描述粗单元内的等效力学响应；在优化过程中结合密度滤波、Heaviside 投影、z 向拔模闭合和固定实体区域约束，以提高结构黑白化程度和制造可行性。为解决大规模 EMsFEM ANN 算子不可直接装配的问题，本文采用 matrix-free 真实算子与装配式 ANN 辅助矩阵相结合的 Krylov-AMG 预条件策略，并在辅助矩阵上附加三维弹性近零空间以改善代数多重网格的粗化质量。工程实现中进一步处理了 PETSc/HYPRE 环境、Slurm 内存配置、DMDA 粗细网格 ownership 对齐、发散求解的优化更新保护以及全细网格 VTK 后处理等问题。最终，该方法成功完成大规模拔模控制臂算例求解，验证了神经网络多尺度有限元与制造约束拓扑优化在大规模工程构件设计中的可行性。

**关键词：** 控制臂；拓扑优化；拔模约束；神经网络形函数；多尺度有限元；PETSc；GAMG；HYPRE；大规模并行计算

## 1 引言

控制臂是车辆悬架系统中的关键承载部件，其结构需要在有限空间内连接衬套、球铰、弹簧安装区等多个功能区域，并在制动、转向、垂向冲击等多种载荷下保持足够刚度和强度。工程上，控制臂通常还需要满足铸造、锻造、拔模、机加工和装配等制造要求。因此，面向控制臂的结构优化不能只追求数学意义上的柔顺度最小，还必须同时关注载荷传递路径、孔系区域实体保留、低密度虚连接、制造闭合方向和大规模计算可实施性。

密度法拓扑优化是工程结构轻量化设计中应用最广泛的方法之一。SIMP 模型通过连续密度变量近似材料插值，使结构优化问题能够转化为大规模约束优化问题。然而，在三维控制臂等复杂构件中，常规低阶有限元若直接细化到制造尺度，会带来极高的自由度数量和内存消耗。另一方面，若仅采用粗网格，又难以表达粗单元内部细观密度分布及其对等效刚度的影响。为平衡效率与精度，本文采用神经网络辅助的多尺度有限元思想：在粗单元尺度求解全局位移场，在粗单元内部使用 5 x 5 x 5 细胞密度描述局部材料分布，并由神经网络预测对应的多尺度形函数，从而近似单元等效刚度。

本文的核心贡献不是提出一个全新的理论模型，而是围绕真实工程尺寸和超算运行条件，构建、调试并验证一套可实际运行的大规模应用流程。该流程特别关注以下问题：

1. 如何在约千万位移自由度和数亿细密度单元下组织 EMsFEM ANN 优化。
2. 如何设置控制臂的工程边界条件，包括 A/B 载荷环、C 固定环和弹簧载荷区。
3. 如何将拔模闭合、Heaviside 投影和强制实体区域整合到优化循环。
4. 如何为 matrix-free EMsFEM ANN 算子构造有效预条件器。
5. 如何在超算 Slurm 环境中避免内存误配、PETSc/HYPRE 不一致和后处理大文件失败。

本文以一个大规模拔模控制臂算例为主线，给出模型定义、算法流程、并行实现、求解器策略和工程调试经验。补充对比算例、图片渲染和最终性能表将在后续版本中完善。

## 2 控制臂算例定义

### 2.1 计算域与网格规模

本文采用的控制臂算例为三维矩形包络域内的结构优化问题。计算域坐标方向定义如下：

- `x` 方向：控制臂长度方向；
- `y` 方向：A/B 两个衬套孔上下分布方向；
- `z` 方向：厚度方向或拔模闭合方向。

默认大规模算例使用以下网格：

| 参数 | 数值 |
|---|---:|
| 粗节点数 `NX` | 325 |
| 粗节点数 `NY` | 186 |
| 粗节点数 `NZ` | 56 |
| 位移自由度 | `3 x NX x NY x NZ = 10,155,600` |
| 粗单元数 | `(NX-1)(NY-1)(NZ-1) = 3,296,700` |
| EMsFEM 细分数 | `sub_n = 5` |
| 单个粗单元细胞数 | `5 x 5 x 5 = 125` |
| 细密度单元数 | `412,087,500` |
| 物理高度 `H` | 0.080000 m |
| 物理长度 `L` | 0.471273 m |
| 物理宽度 `W` | 0.269091 m |

其中物理长度和宽度由网格比例和指定高度自动给出：

```text
L = (NX - 1) / (NZ - 1) * H
W = (NY - 1) / (NZ - 1) * H
```

### 2.3 设计域、非设计域和强制实体区域

优化变量定义在细密度网格上，而不是粗位移网格上。每个粗单元包含 `5 x 5 x 5` 个细密度细胞。代码中用 `mask` 区分不同区域：

```text
mask = 0: 非设计区，保持 void_density
mask = 1: 可优化设计区
mask = 2: 强制实体区，rho = 1
```

非设计区来自控制臂外轮廓、孔洞和内部清空区。强制实体区包括：

1. A 环实体圈；
2. B 环实体圈；
3. C 环实体圈；

## 3 边界条件与载荷工况

### 3.1 位移边界条件

本文算例采用 C 环固定。选中的 C 环节点满足：

```text
u_x = 0
u_y = 0
u_z = 0
```

C 固定环的中心为：

```text
C = (0.78L, 0.5W, 0.5H)
```

固定节点选择条件近似为：

```text
|r_xy - r_C| <= tol
|z - 0.5H| <= axial
```

其中：

```text
r_C = 0.10 * min(L, W) = 0.026909 m
tol = 0.75 * hmax
axial = max(0.15 * min(L, W), 2 hmax) = 0.040364 m
```

由于结构总厚度为 `H = 0.08 m`，`axial ≈ 0.040364 m`，因此 C 环固定约束基本贯穿整个厚度方向。工程上可以理解为：C 环孔周的圆周节点作为夹持或安装支承，被完全固定。

### 3.2 A/B 载荷环的力与力矩分布

A 和 B 位置施加的不是单个节点集中力，而是通过孔周节点分布的等效节点力。目标是使孔周节点力满足指定合力和合力矩：

```text
wrench = [Fx, Fy, Fz, Mx, My, Mz]
```

对孔周每个节点，其节点力表示为位置相关的线性形式。程序先组装一个 `6 x 6` 的分布矩阵，再求解得到分布参数，使总合力和总力矩匹配给定 wrench。这样比单点集中力更符合衬套或连接件的载荷传递形式，也能避免单节点奇异载荷导致局部拓扑畸变。

弹簧安装区施加竖向向下载荷：


该载荷均匀分配到弹簧安装区选中的节点上。若 `include_spring_load=true`，则每个工况都会包含对应大小的弹簧向下力。

若提交脚本中使用：

```bash

```

则只计算第二组工况，且权重为 1。第二组工况中：

```text
A 环主载荷方向接近 +z
```

```text

case 2: 0.50
case 3: 0.25
```

这使优化结果同时考虑多个典型受载状态，避免结构只适配单一路径。

## 4 优化模型

### 4.1 目标函数

本文采用柔顺度最小化作为目标。单工况下：

```text
minimize: C = f^T u
```

其中 `f` 为外载荷向量，`u` 为求解得到的位移向量。多工况时，目标函数为加权柔顺度：

```text
C = sum_i w_i f_i^T u_i
```

其中 `w_i` 为工况权重。

### 4.2 约束条件

主要约束包括：

1. 体积分数约束：

```text
mean(rho_design over active design cells) <= 0.30
```

2. 设计变量上下界：

```text
rho_min <= rho <= 1
rho_min = 1e-3
```

3. 非设计区约束：

```text
rho = void_density = 0.02
```

4. 强制实体区约束：

```text
rho = 1
```

5. Heaviside 投影：

```text
rho_phys = H_beta(rho_filtered, eta)
```

其中 `eta=0.5`，`beta` 随迭代逐步增大。

6. z 向拔模闭合约束：

```text
rho_phys(x, y, z) 经过 z-column closure 后用于分析
```

### 4.3 SIMP 材料插值

材料刚度采用 SIMP 插值：

```text
E(rho) = Emin + (1 - Emin) * rho^penal
```

当前算例参数为：

```text
E0 = 2.1e11
penal = 3.0
Emin = 1e-6
```

这里需要区分两个量：

- `Emin=1e-6` 是材料最小刚度比例，用于避免完全奇异；
- `rho_min=1e-3` 是设计变量下界，用于 OC 更新稳定性。

二者不应混淆。

### 4.4 密度滤波与 Heaviside 投影

优化过程中先对设计变量进行密度滤波，再进行 Heaviside 投影。当前参数为：

```text
filter_radius = 1.5
heaviside_eta = 0.5
beta_initial = 1
beta_max = 16
beta_interval = 100
```

因此 500 步算例的 beta 计划为：

| 迭代区间 | beta |
|---|---:|
| 1--100 | 1 |
| 101--200 | 2 |
| 201--300 | 4 |
| 301--400 | 8 |
| 401--500 | 16 |

本文在调试中发现，`beta=8` 后线性系统明显变难，KSP 迭代数会显著上升。因此，beta 增长速度不能过快。过早提高到 `beta=16` 或 `beta=32` 虽然有助于黑白化，但会使低密度区域和窄连接区域造成更强的病态性，进而影响求解收敛。

### 4.5 z 向拔模闭合

为了提高结构在 z 方向上的制造可行性，本文在物理密度上施加 z 向闭合。直观上，对同一 `(x, y)` 柱内的密度进行上下方向的信息传播，使局部孤立孔洞或不连续薄层被闭合。这一处理与 Heaviside 投影结合后，有助于形成更容易拔模或至少更连贯的厚度方向结构。

## 5 神经网络多尺度有限元

### 5.1 粗单元与细胞密度

传统 H8 有限元中，每个粗单元内材料均匀或采用简单插值。本文中，每个粗单元被细分为 `5 x 5 x 5` 个细胞。细胞密度用于描述单元内部细观材料分布。若直接为每个细胞建立全局自由度，计算规模会远超当前超算任务的合理范围。因此，本文采用 EMsFEM ANN 思路，在粗节点位移自由度上求解全局问题，同时通过神经网络预测单元内部形函数来近似细观影响。

### 5.2 神经网络形函数

神经网络输入为局部细胞材料分布，输出为单元内部多尺度形函数。计算单元刚度时，程序先根据当前密度构造细观材料参数，再调用 ANN 得到修正形函数，并与细观刚度积分组合得到粗单元等效刚度。

这一做法的应用优势是：

1. 维持全局自由度在粗节点尺度；
2. 保留粗单元内部 `5 x 5 x 5` 细胞密度表达能力；
3. 避免每一步显式求解大量局部细观边值问题；
4. 使控制臂这种大规模三维构件能够在超算上完成多步优化。

### 5.3 matrix-free 真实算子

真实 EMsFEM ANN 刚度算子采用 matrix-free 形式。也就是说，全局矩阵 `A` 并不完整装配为稀疏矩阵，而是在 MatMult 时根据单元刚度缓存执行局部乘法并累加。这降低了真实算子存储压力，但也带来一个问题：许多 PETSc 预条件器，尤其是 AMG，需要装配矩阵图和数值矩阵，不能直接高效作用于 shell operator。

因此，本文采用“真实算子 matrix-free，预条件矩阵装配”的折中策略。

## 6 线性求解器与预条件策略

### 6.1 基本线性系统

每个工况下求解：

```text
A(rho) u = f
```

其中：

- `A` 是 matrix-free EMsFEM ANN 真实算子；
- `u` 是粗节点位移；
- `f` 是 A/B/Spring 分布载荷。

由于 Heaviside 投影、低密度区域、强制实体区和 ANN 近似形函数共同作用，矩阵条件数在优化后期显著增加，尤其当 beta 从 4 增加到 8 后，KSP 迭代数会明显上升。

### 6.2 辅助预条件矩阵

本文构造装配式 ANN auxiliary matrix `P`。求解时使用：

```text
KSP: FGMRES
operator A: matrix-free EMsFEM ANN shell operator
preconditioner matrix P: assembled ANN auxiliary matrix
PC: GAMG or HYPRE BoomerAMG
```

即：

```text
KSPSetOperators(ksp, A, P)
```

这样保留了真实算子的 matrix-free 优势，同时让 AMG 能够基于装配矩阵 `P` 构造层次结构。

### 6.3 弹性近零空间

对于三维弹性问题，GAMG 需要知道刚体平移和转动模态，才能更合理地构造粗层空间。本文在 ANN auxiliary matrix 上附加 6 个三维弹性近零空间模式：

```text
3 个平移模态
3 个转动模态
```

固定节点处近零空间分量被跳过，以避免与 Dirichlet 约束冲突。该处理显著改善了 GAMG 对弹性类问题的适配性。

### 6.4 预条件器配置

当前推荐配置为：

```text
EMS_PC_TYPE=aux_ann_gamg
GAMG_PROFILE=xstrong
KSP_TYPE=fgmres
PC_TYPE=gamg
KSP_RTOL=1e-3
KSP_MAX_IT=2000
KSP_GMRES_RESTART=300
节点数: 48
```

备用配置为：

```text
EMS_PC_TYPE=aux_ann_hypre
HYPRE_PROFILE=xstrong
```

GAMG 强档适用于当前已经跑通的主线。若 beta 后期仍出现 KSP 打满迭代，HYPRE BoomerAMG 强档可作为对照或替代。

## 7 并行实现与超算运行

### 7.1 PETSc DMDA 粗细网格

本文使用两个关键 DMDA：

1. `uda`：粗节点位移场，尺寸为 `NX x NY x NZ`，每个节点 3 自由度；
2. `fda`：细密度场，尺寸为 `(NX-1)sub_n x (NY-1)sub_n x (NZ-1)sub_n`，每个细胞 1 个密度自由度。

早期调试发现，若让 PETSc 分别自动切分 `uda` 和 `fda`，二者 ownership 不一定严格对齐。由于 EMsFEM ANN 的单元刚度缓存会按粗单元范围访问细密度 ghost 区，这种错位会造成并行越界读、矩阵错误或求解异常。

最终实现采用：

```text
先创建 uda
读取 uda ownership ranges
由粗节点 ownership 推导粗单元 ownership
再乘 sub_n 得到 fda ownership
```

并加入 ghost coverage check，使错误分区在运行初期显式报错，而不是在 KSP 或 AMG 阶段表现为随机不收敛。

### 7.2 节点数与进程网格

当前脚本默认：

```text
48 nodes
28 tasks per node
total ranks = 1344
```

对 `325 x 186 x 56` 粗节点网格，自动进程网格倾向于接近：

```text
px x py x pz = 24 x 14 x 4
```

该分区比 32 节点时更均衡，能降低每 rank 的 ANN/K-cache 工作量和辅助矩阵存储压力，同时仍保留足够局部自由度供 AMG 粗化。

## 9 工程调试与问题修复记录

本节总结从初始失败到最终可跑过程中遇到的关键问题。这些问题虽然不一定属于理论创新，但对大规模应用能否真正运行至关重要。

### 9.1 PETSc/HYPRE 环境

早期运行中，代码需要使用 HYPRE 或 PETSc GAMG 预条件器，但 PETSc 环境并不一定带 HYPRE。解决方法是确认：

```bash
grep -i hypre $PETSC_DIR/$PETSC_ARCH/lib/petsc/conf/petscvariables
```

并在必要时重编 PETSc：

```bash
./configure \
  --with-cc=mpicc \
  --with-cxx=mpicxx \
  --with-fc=mpif90 \
  --with-64-bit-indices=1 \
  --with-debugging=0 \
  COPTFLAGS="-O3" \
  CXXOPTFLAGS="-O3" \
  FOPTFLAGS="-O3" \
  --download-hypre
```

同时需要注意，`make check` 中 2 MPI 进程失败并不一定说明 HYPRE 编译失败，也可能是 UCX/OFI/MPI 启动环境问题。

### 9.2 Slurm 内存配置异常

曾出现 Slurm 作业 `ReqMem=4M`、节点 `RealMemory=1` 的异常显示，导致作业很快失败。该问题不是代码物理模型错误，而是 Slurm 资源配置或节点内存上报异常。脚本中使用：

```bash
#SBATCH --mem=0
#SBATCH --exclusive
```

并增加环境输出与节点检查，减少误判。

### 9.3 预条件器从弱到强

最初的 block Jacobi 对千万自由度控制臂不足以支撑后期优化。随后引入 auxiliary matrix：

1. H8 辅助矩阵 + HYPRE；
2. ANN auxiliary matrix + HYPRE；
3. ANN auxiliary matrix + PETSc GAMG；
4. GAMG 上附加三维弹性近零空间；
5. 增加 `strong` 和 `xstrong` 预条件器 profile。

最终主线采用：

```text
FGMRES + ANN auxiliary matrix + PETSc GAMG
```

### 9.4 粗细 DMDA ownership 错配

这是最关键的并行 bug。`uda` 是粗节点位移 DMDA，`fda` 是细密度 DMDA。如果二者分别自动分区，则某些 rank 在构造 EMsFEM 单元刚度缓存时，可能访问超出 `fda` ghost 区的细密度数据。

修复方法：

```text
由 uda ownership 推导 coarse element ownership
再乘 sub_n 得到 fda ownership
```

并增加 ghost coverage check。该修复显著提高了并行运行的确定性。

### 9.5 sensitivity 中固定长度数组

旧代码中存在：

```cpp
PetscReal energies[125];
```

这只适用于 `sub_n=5`。若后续使用 `sub_n=10`，需要 1000 个细胞能量，会造成越界风险。修复为动态 vector 后，代码对不同 `sub_n` 更安全。

### 9.6 线性求解发散后的优化更新

早期逻辑中，KSP 发散后仍可能进入灵敏度和更新流程，导致无效位移场污染设计。修复后：

1. 若首个线性求解发散，直接拒绝更新；
2. 若已有 last good design，发散工况跳过灵敏度；
3. 恢复上一 accepted 设计；
4. 缩小 move 后重试；
5. 连续多次最小 move 拒绝后提前停止并恢复最后可接受设计。

### 9.7 Heaviside beta 过快导致后期 KSP 变慢

在一次 500 步运行中，结构在 `beta=4` 时一直稳定，KSP 约 95 次；当第 301 步 `beta` 跳到 8 后，KSP 迭代数快速升至 277，并出现 `DIVERGED_ITS=300`。这说明后期投影变硬会显著增加线性系统病态性。

解决方法：

1. beta 每 100 步翻倍；
2. 避免过早到 32；
3. 提高 KSP 最大迭代数；
4. 增强 GAMG/HYPRE profile；
5. 修复拒绝步中 raw volume target 被错误重置到 0.3 的问题。

### 9.8 VTK 写出

大规模全细网格 VTK 文件非常大，不适合在主优化程序中 root gather 写出。最终采用：

```text
优化阶段：写 PETSc binary checkpoint
后处理阶段：独立脚本转换全细网格 VTK
```

这既避免优化阶段内存爆炸，也方便在本地或超算后处理节点上生成可视化文件。

## 10 讨论

### 10.1 低密度连接与阈值敏感性

在控制臂拓扑优化中，常见现象是低阈值下结构看似连通，而高阈值下局部断开。这通常与以下因素有关：

1. SIMP 的最小刚度使低密度区域仍能传递少量刚度；
2. Heaviside 投影前期较软，中间密度较多；
3. 滤波半径会产生过渡区；
4. z 向闭合会增强厚度方向连续性，但不保证所有高阈值等值面连通；
5. 后期 beta 增大使中间密度快速转向 0 或 1，可能暴露细弱连接。

因此，不能只用 `rho=0.1` 的等值面判断结构可制造性，也不能只用 `rho=0.8` 的断开就认定优化失败。更合理的做法是同时检查：

```text
rho = 0.1, 0.3, 0.5, 0.8, 0.95
```

并结合体积分数、柔顺度、连通性和制造后处理判断。

### 10.2 单工况与多工况

单工况 `LOAD_CASE=2` 有利于先验证模型、求解器和后处理流程。但工程控制臂通常需要多工况载荷。默认 `LOAD_CASE=0` 的三工况加权优化更接近工程应用，但计算代价约为单工况的三倍，且后期 KSP 更容易遇到难收敛问题。

因此，推荐路线为：

1. 单工况完成流程打通；
2. 单工况增强预条件器；
3. 三工况短步数验证；
4. 三工况 500 步生产运行。

### 10.3 ANN 多尺度近似的工程意义

ANN 多尺度形函数的价值在于允许粗单元内部保留细密度结构，同时避免每步求解大量局部边值问题。对于控制臂这类几何尺寸大、局部细节重要的工程构件，这种方法具有较好的应用潜力。

但它也带来新的挑战：

1. ANN 预测误差可能影响等效刚度的正定性和光滑性；
2. 后期低密度和硬投影可能放大矩阵病态性；
3. 预条件器不能只依赖真实 matrix-free 算子，需要辅助矩阵；
4. 需要严格处理粗细网格并行分区一致性。

## 11 结论

本文围绕一个大规模拔模控制臂算例，建立并验证了一套神经网络多尺度有限元拓扑优化流程。主要结论如下：

1. 采用 `5 x 5 x 5` 细密度细胞和 ANN 形函数预测，可以在粗位移自由度规模下表达单元内部细观材料分布，使约 4.12 亿细密度单元的控制臂优化成为可能。
2. 控制臂工程边界条件可通过 A/B 孔周分布 wrench、C 环全约束和弹簧竖向载荷实现，比单节点集中力更适合孔系结构。
3. 强制实体区、非设计区、密度滤波、Heaviside 投影和 z 向拔模闭合共同构成制造约束流程，有助于得到更连通、更可制造的结构。
4. 对 matrix-free EMsFEM ANN 算子，直接使用常规预条件器不足以支撑大规模后期优化；装配 ANN auxiliary matrix，并使用 GAMG/HYPRE AMG 预条件，是可行的工程策略。
5. 三维弹性近零空间、粗细 DMDA ownership 对齐、KSP 发散保护和强预条件器 profile 是最终算例跑通的关键工程修复。
6. 后期 Heaviside beta 增大时线性系统明显变难，应采用温和 continuation 和足够强的预条件器，而不是单纯提高迭代步数。

后续工作将补充多工况对比、不同预条件器性能表、最终结构的多阈值连通性分析、位移/应力云图以及与传统 H8 matrix-free 优化结果的定量比较。
